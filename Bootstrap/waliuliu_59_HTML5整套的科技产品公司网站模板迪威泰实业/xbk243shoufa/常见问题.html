  • IP Camera连接NVR移动侦测报警无效？

    可能原因：IP Camera没有打开移动侦测报警。

    解决办法：检查IP Camera 移动侦测报警是否启用，移动侦测区域是否设置，移动侦测灵敏度是否设置太低（灵敏度从1-5。1最低5最高）。

    可能原因：NVR没有打开移动侦测报警，

    解决办法：检查NVR移动侦测报警是否启用。


    阅读详细

  • IP Camera连接NVR不出图像？

    可能原因：IP Camera和NVR不在同一网段

    解决办法：修改IP Camera或者NVR的IP地址，使其在同一网段。

    可能原因：IP Camera与局域网内某一设备的IP地址冲突。

    解决办法：修改IP Camera的IP地址。重新连接NVR。

    可能原因：在NVR上添加IP Camera时没有输入正确的IP Camera登陆密码。

    解决办法：添加IP Camera时输入正确的IPC登陆密码。（注：NVR添加IP Camera时会有一个添加设备提示框，其实有要求输入用户名和密码，这里的用户名和密码是IP Camera的用户名和密码，不是NVR的用户名和密码）


    阅读详细

  • 启用DHCP后，设备无法自动获取到IP地址？

    可能原因：路由器DHCP服务器没有启用。

    解决办法：DHCP是Dynamic Host Configuration Protocol的简称，用于给局域网设备自动分配IP地址。在启用DHCP后，要求有一个正常运行的DHCP服务器存在。一般是用路由器实现DHCP服务器功能，当设备无法自动获取到IP时，需要检查一下路由器有没有启用DHCP功能。


    阅读详细

  • 升级设备成功后，重新登录设备出现排版不正常情况。

    可能原因：IE排版有改动，重新登录设备调用缓存中的数据，造成排版不正常

    解决方法：打开浏览器，点击“工具”菜单，选择“Internet 选项”，在“Internet 临时文件”中单击“删除文件”按钮，删除IE缓存。


    阅读详细

  • 忘记了密码该怎么办?

    解决方法：在IP Camera后板或接口线上有一个【RESET】按钮，在上电状态下，按住复位键1-2秒，松开1-2秒，重复三次，设备会恢复出厂值，默认IP：192.168.1.88 默认用户名密码：admin/admin

    注意：非专业人员请不要按RESET键。复位后，所有参数将恢复到出厂值（除网络物理地址外）。

  • 监听时无声音

    可能原因1：没有接入音频输入？

    解决方法：检查主机的音频连接。

    可能原因2：IP Camera没有打开相应通道的音频选项？

    解决方法：检查IP Camera音频参数设置-音频属性-音频开关，看是否打开了音频，预览界面对讲和监听是否打开。

  • 图像显示颜色不正常(呈绿色或其它色)

    可能原因：视频参数做过修改

    解决方法：检测视频参数中色调以及白平衡，建议使用自动白平衡，非专业人事请勿使用手动白平衡。


    阅读详细

  • 为什么视频图像不流畅？

    可能原因1：IP Camera设置的视频帧率太低，造成图片非实时。

    解决方法：将视频帧率参数调高，PAL制式最高25FPS，NTSC制式最高30FPS。

    可能原因2：同一时间太多用户连接设备，造成设备CPU负载过高。

    解决方法：可以关闭部分客户端或设置IP Camera减少帧率或码流。

    可能原因3：网络带宽太低，数据拥塞，造成丢包。

    解决方法：可以减低视频的帧率或者降低码率，减少网络负担。

    可能原因4：服务器硬件配置过低，造成解码不足

    解决方法：高清设备界面对显卡，CPU，内存都有一定要求，检测服务器CPU已经内存使用情况，采用设备次码流进行解码，减低主码流的码流以减少服务器负担。


    阅读详细

  • Device Search以及CMS搜索软件搜索不到设备

    可能原因1：由于Device Search以及CMS软件使用多播协议进行跨网段搜索设备网络信息，而防火墙是不允许多播数据包通过的，所以导致搜索不到设备的网络信息。

    解决方法：将防火墙关闭。

    可能原因2：设备与服务器不在同一个局域网

    解决方法：检测网络，保证设备与服务器在统一个局域网中
